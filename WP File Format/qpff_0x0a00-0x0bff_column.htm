Quattro Pro File Format

Column Records - 0x0A00-0x0BFF

The following are column records, which span from 0x0A00 to 0x0BFF.

0x0A01

BOC : Beginning of column

The data in each column (cells are stored in column order) begins with a BOC record. The position of each BOC record is stored in the filePos field of the BLOCK record.

[record type = 2561 (0x0a01)]

[body length = 10]

[column] column number for all cell records to follow until EOC

[first row] index of first row

[last row] index of last row

0x0A02

EOC : End of column

Marks the end of a column begun by corresponding BOC record

 

[record type = 2562 (0x0a02)]

[body length = 8]

{prev end} file position of previous EOC (or corresponding beginning of column record if this is the first column)

{nested end} always zero

0x0A03

COLUMN_FORMAT

[record type = 2563 (0x0a03)]

[body length = 2]

[index] -1 = use sheet default

0x0A04

COLUMN_WIDTH

UNIMPLEMENTED

[record type = 2564 (0x0a04)]

[body length = variable]

QUIP width column width

0x0A05

COLUMN_INFO

UNIMPLEMENTED

[record type = 2565 (0x0a05)]

[body length = 2]

[flags]

bits 0-3: hidden

0 = normal

1 = hidden

2 = very hidden (only showable through Microsoft Visual Basic routine)

15 = use sheet default

bits 4-7: outline level

bit 8: collapsed

1 = collapsed in outlining

bits 9-15: reserved
