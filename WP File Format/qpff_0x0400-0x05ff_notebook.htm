Quattro Pro File Format

Notebook Records - 0x0400-0x05FF

The following are notebook records, which span from 0x0400 to 0x05FF.

0x0401

BON: Beginning of Notebook 

[record type = 1025 (0x0401)]

[body length = 24]

QRANGE3D dimension, 3D range of cells used in the notebook

[sheet limit] maximum number of sheets in the notebook

[column limit] maximum number of columns per sheet

{row limit} maximum number of rows per column

status: Reviewed

implemented load and save: Yes

0x0402

EON: End of Notebook 

[record type = 1026 (0x0402)]

[body length]

{prev end} file position of beginning of notebook (BON) record (since single notebook per file)

{nested end} file position of the last end of sheet record (0 if no sheets)

status: Modified

implemented load and save: Yes

0x0405

EXTERNAL_NAME

IMPLEMENTATION NOT COMPLETE

[record type = 1029 (0x0405)]

[body length = variable]

[format]

bit 0: 1 = name is a build-in name

bit 1:

0 = manual DDE links

1 = automatic DDE links

bit 2: wantPict

bit 3: 1 = record stores OLE standard document name

bit 4: 0 = DDE link

bits 5-14: clipboard format for which the DDE Advise succeeded

bit 15: 0 = SSREF follows name instead of nameDef

{idx}

[niOwner] the notebook which the name belongs to

QSTR_LABEL name

SSREF reference of the name
 
 

status: Proposed

implemented load and save: Partially (DDE and OLE links not done)

0x0406

NAME _LIST: name (including user-defined functions)

IMPLEMENTATION NOT COMPLETE

[record type = 1030 (0x0406)]

[body length = variable]

[count]

[record length]

[name]

bit 0: 1 = name is hidden

bit 1: 1 = name is a function

bit 2: 1 = name is a Visual Basic procedure

bit 3: 1 = name is a function or command name on a macro sheet

bit 4: 1 = name contains a complex function; a function which returns an array (for eg. TREND, MINVERSE), contains a ROW or COLUMN function, or contains a user-defined function

bit 5: 1 = name is a built-in name; the def field will have a count of 1, with the one byte definition as follows

1 = Auto_Open

2 = Auto_Close

3 = Extract

4 = Database

5 = Criteria

6 = Print_Area

7 = Print_Titles

8 = Recorder

9 = Data_Form

10 = Auto_Archive

11 = Auto_Deactivate

12 = Sheet_Title

bits 6-11: function group index

bit 12: 1 = name refers to binary data. These names can be created only by calling the xlDefineBinaryName function from the Microsoft Excel C API. The first byte is length of the name, which is followed by the name string. Following the name string is the data to which the name refers

bits 13-15: reserved

[format]

bit 0: implicit. This bit is set if name is implicit from a label

bit 1: undoable

bits 2-3: exeType

bit 4: inLoop

bit 5: inLoad (always 0)

bit 6: defined

bit 7: nameDef (always 0 in version 9)

bits 8-15: unused

[macro key] keyboard shortcut, if any, if 3rd bit is set in name field

QSTR_LABEL name text

SSREF reference

QSTR_LABEL description

QSTR_LABEL custom menu text

QSTR_LABEL help topic text

QSTR_LABEL status bar text
 
 

status: Reviewed

implemented load and save: Yes

required memory changes done: No (Microsoft Excel compatibility not added)

0x0461

SCENARIO_TABLE_BEGIN
 
 

[record type = 1121 (0x0461)]

[body length = variable]

SSRANGE captureRange active capture range

[flag]

bit 0: isNew rue if allocated but not added

bit 1: isDirty indicates that changes have been made

bit 2: isWorking this is the current selected scenario

bit 3-4: captureMode type of capture notebook, page, block

bit 5: autoResult auto calc result cells on change cell input

bit 6: trackData true to track data move/insert/deletes

bit 7: leftLabels use labels to the left in reports

bit 8: topLabels use labels above in reports

bit 9: explicitRange range has been explicitly set

bits 10-15: reserved

QSTR_LABEL name name of this table

QSTR_LABEL curName value in scenario combobox
